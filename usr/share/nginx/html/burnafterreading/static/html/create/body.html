F.A.Q.

Why would I use this?

When you send people passwords or private links via email or chat, there are copies of that information stored in many places. If you use a one-time link instead, the information persists for a single viewing which means it can't be read by someone else later. This allows you to send sensitive information in a safe way knowing it's seen by one person only. Think of it like a self-destructing message.

How long do you keep non-viewed encrypted secret message?

We keep encrypted secret message for up to 24 hours from the moment of creation. After that they are deleted automatically forever.

What is the maximum message size?

The maximum size is 1MB.

Can I retrieve a encrypted secret message that has already been read?

Nope. We display it once and then delete it. After that it's gone forever.

Why should I trust you?

We can't do anything with your information even if we wanted to. All data before being sent to our server is encrypted in your browser using the Web Crypto API. We don't store the encryption key so we can never know the secret message is because we can't decrypt it. To perform authenticated encryption is used AES in Galois/Counter Mode, as described in [NIST SP 800-38D],with a random 96-bit initialization vector and random 256-bit encryption key. Beside all that, the code is opensource so you can review the code and/or run your own instance.

F.A.Q.

What if I forgot to copy the shared link or decryption key?

You need to create a new shared link. We can't restore it for you.

How long will the encrypted secret message be available?

The encrypted secret message will be available for 24 hours from the moment of creation or until first attempt it's decrypt.

What happens when I burn a encrypted secret message?

Burning a encrypted secret message will delete it before it has been read. If you send someone a shared link and burn the encrypted secret message before they view it, they will not be able to read it.
